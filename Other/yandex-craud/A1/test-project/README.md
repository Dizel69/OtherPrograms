# Задание А1

Все команды применять в той же директории, где и Dockerfile и docker-compose файлы.

## Запуск контейнера через Dockerfile

### Сборка контейнера
Для того чтобы запустить контейнер его необходимо сначала собрать:

```bash
docker build -t my-web-app:latest .
```

>, где -t - тег образа, - . директория, где лежит Dockerfile

### Запуск контейнера
Чтобы запустить контейнер из готового образа необходимо выполнить команду:
```bash
docker run -d -p 8080:80 --name my-web-container my-web-app:latest
```

>, где -d — запускает контейнер в фоне, -p 8080:80 — пробрасывает порт 8080 хоста на порт 80 контейнера, --name — задаём имя контейнера.

Теперь если в браузере зайти в http://localhost:8080 можно увидеть сообщение "Hello from Dizel". 

## Запуск контейнера через docker-compose
Запуск контейнера через docker-compose:
```bash
docker-compose up -d
```

## Ответ на вопрос Как можно доставить твой index.html в контейнер, не пересобирая образ?
Можно использовать volumes в Docker:

```bash
docker run -d -p 8080:80 -v $(pwd)/index.html:/usr/share/nginx/html/index.html --name my-web-container my-web-app:latest
```
Таким образом: 
- Файл на хосте монтируется внутрь контейнера.
- Любые изменения файла на хосте сразу будут видны в контейнере.
